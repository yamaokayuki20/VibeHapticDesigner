# 仕様書：リアルタイム触覚生成モジュール（完全版 / MVP Androidデモ）

---
# 1. システムアーキテクチャ構成

## 1.1 目的
自然言語から「ビジュアル + 触覚特性 + 物理特性 + 音響特性」を持つオブジェクトを生成し、
仮想デスク上での衝突・摩擦イベントに応じてリアルタイムに触覚および音を合成再生する。

---
## 1.2 全体構成

### Androidクライアント
- 自然言語入力UI
- 生成結果表示（画像 / メッシュ）
- 触覚4軸パラメータ編集（スライダ）
- プレビュー再生（衝突サンプル + 音）
- 仮想デスクUI
- 衝突検出・状態判定
- 触覚動的生成エンジン
- 音動的生成エンジン
- ローカルキャッシュ保存

### Backend（AI生成）
- プロンプト解釈
- 材質推定
- JSON構造出力
- 画像生成またはアセット選択
- サウンドプリセット推定

---
## 1.3 データフロー

1. プロンプト入力
2. POST /v1/objects/generate
3. JSON + 画像返却
4. 編集画面表示
5. プレビュー再生
6. PLAY → デスク配置
7. 衝突・摩擦 → 動的生成 → 再生

---
# 2. データモデル設計

## 2.1 触覚4軸（0.0〜1.0）

- roughness : 表面粗さ
- solidness : 液体〜固体
- hardness  : やわらかさ〜硬さ
- heaviness : 軽さ〜重さ

# 4. 触覚動的生成アルゴリズム

## 4.1 Pave算出

Pave = (P1 + P2) / 2

---
## 4.2 状態判定

IMPACT:
- 接触開始
- 速度または加速度閾値超過

FRICTION:
- 接触継続
- 接線速度閾値超過

IDLE:
- 非接触

---
## 4.3 疑似コード

```pseudo
Pave = (P1 + P2) / 2

if contactJustStarted and (v_rel > V_TH or a_rel > A_TH):
    state = IMPACT
elif isContact and tangentialSpeed > V_FRICTION_TH:
    state = FRICTION
else:
    state = IDLE
```

---
## 4.4 パーリンノイズ生成方針

- roughness ↑ → 高周波ノイズ増加
- hardness ↑ → アタック鋭く
- heaviness ↑ → 低周波増加
- solidness ↓ → 減衰長く
